报错处理

Your local changes to the following files would be overwritten by merge

原因是拉取远程仓库的时候本地版本比远程仓科还要新
解决方案:
根据是否要保存本地修改，有以下两种解决方案

1 保留修改


执行以下三条命令
git stash #封存修改
git pull origin master
git stash pop #把修改还原

2 不保留修改


git reset --hard
git pull origin master

封存和回退

cannot lock ref 'refs/remotes/origin/master': unable to resolve reference 'refs/remotes/origin/master': reference broken

解决方法

**第一步：**删除到.git目录下的master文件
rm ./.git/refs/remotes/origin/master
**第二步：**抓取远程仓库最新代码
git fetch
**第三步：**远程分支内容合并到本地master分支下
git merge origin/master
至此，成功将远程的最新代码，pull到本地

传送门

Merge branch 'master' of github.com:xiaoRuMiLi/rapairStaff # Please enter a commit message to explain why this merge is necessary, # especially if it merges an updated upstream into a topic branch. # # Lines starting with '#' will be ignored, and an empty message aborts # the commit.

这时候要求输入一段命令

即：请输入提交消息来解释为什么这种合并是必要的
git 在pull或者合并分支的时候有时会遇到这个界面。可以不管(直接下面3,4步)，如果要输入解释的话就需要:
1.按键盘字母 i 进入insert模式
2.修改最上面那行黄色合并信息,可以不修改
3.按键盘左上角"Esc"
4.输入" :wq",冒号+wq,按回车键即可
